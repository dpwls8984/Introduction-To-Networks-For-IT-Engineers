### 용어 정리

1. OSI 7계층 모델
2. TCP/IP 4계층 모델
3. 네트워크 인터페이스 카드 (NIC)
    1. PC나 서버에 박혀 있는 물리적인 랜 카드
    2. 제조사가 부여한 MAC 주소가 적혀있다.
4. MAC 주소 
    1. 랜카드나 와이파이 모듈 등 네트워크 기기에 제조사에서 부여한 48비트(12자리 16진수)의 고유한 물리적 주소
5. 인캡슐레이션과 디캡슐레이션
6. MSS & MTU
    1. 둘 다 데이터 크기를 지칭한다. 
        1. MTU값은 2계층의 데이터 값, MSS는 4계층에서 가질 수 있는 최대 데이터 값
        2. 2계층에서 2계층 헤더 크기를 제외한 데이터 크기를 MTU 라고 함.
    2. MSS(Maximum Segment Size): 네트워크에서 수용할 수 있는 크기를 역산정해 데이터가 4계층으로 내려올 때 적절한 크기로 쪼개질 수 있도록 유도한 값
    3. MTU(Maximum Transmission Unit): 네트워크에서 한 번에 보낼 수 있는 데이터 크기

### OSI 모델과 TPC/IP 모델은 양립할 수 없는 관계인가? 무슨 관계인가?

- OSI 7계층 모델을 가지고 좀 더 실무에 적합한 구조인 TPC/IP 모델을 만들어낸 것.
- OSI 7계층 모델은 좀 더 이론적이고, TPC/IP 4계층 모델은 현재 사용하는 모델

### OSI 7계층 이해하기

```yaml
- 1~4계층: 데이터 플로 계층 / 하위 계층
    - 데이터를 잘 쪼개 보내고 받는 역할을 중점적으로 고려한다
    - 상위 계층 입장에선 하위 계층으로 데이터를 내려보내기만 하면 알아서 잘 데이터를 쪼개 정보를 붙여 목적지까지 전달한다
- 5~7계층: 애플리케이션 계층 / 상위 계층
```

> 1계층 (피지컬 계층)
> 
- 물리적 연결과 관련된 정보를 정의한다. 주로 전기 신호를 전달하는 데 초점이 맞춰져있다.
- 들어온 전기 신호를 그대로 잘 재생성하여 전달하는 것이 목적이다.
- 1계층에는 주소의 개념이 없으므로 전기 신호가 들어온 포트를 제외하고 모든 포트에 같은 전기 신호를 전송한다. (출발지 & 목적지 구분 X)
    - 2계층에 올라가면서 자신에게 온 전기 신호가 아니라면 그 포트는 해당 전기 신호를 버린다.

> 2계층 (데이터 링크 계층)
> 
- 전기 신호를 모아 알아볼 수 있는 데이터 형태로 처리한다.
- 주소 정보를 정의하고 정확한 주소로 통신이 되도록 한다.
- 출발지, 도착지 주소 확인하고 데이터 처리 수행한다.
- 주소 체계가 생기는 계층이기 때문에 여러 통신이 한꺼번에 이루어지는 것을 구분하기 위한 기능이 주로 정의된다.
- 무작정 데이터를 보내는 게 아니라, 데이터를 받는 사람이 현재 데이터를 받을 수 있는지 확인하는 작업부터 이루어진다.
- 물리 계층에서 받은 비트(Bit) 데이터를 프레임(Frame) 단위로 묶어 처리하며, 대표 장비로 스위치(Switch)와 브리지가 있고 주소 체계로 MAC 주소를 사용한다.
    - MAC 주소란?
        - 랜카드나 와이파이 모듈 등 네트워크 기기에 제조사에서 부여한 48비트(12자리 16진수)의 고유한 물리적 주소
        - 네트워크 카드(NIC)의 고유한 숫자로, 변경 불가하다.
- _NIC 동작 방식_
    1. 전기 신호를 데이터 형태로 만든다
    2. 목적지 MAC 주소와 출발지 MAC 주소를 확인한다
    3. NIC의 MAC 주소를 확인한다
    4. 목적지 MAC 주소와 NIC가 갖고있는 MAC 주소가 맞으면 데이터를 처리하고, 다르면 데이터를 폐기한다.
        1. _위 4단계의 동작 방식은 인캡슐레이션, 디캡슐레이션 중에 무엇을 설명한 것인가?_

> 3계층 (네트워크 계층)


```yml
IP 주소를 통해 원거리 시스템간 통신이 가능하게 함
```

- IP 주소같은 논리적인 주소가 정의된다.
- 데이터 통신할 때 필요한 것이 2계층의 **물리적인** MAC 주소와 3계층의 **논리적인** IP 주소이다.
- IP 주소는 사용자가 환경에 맞게 변경할 수 있다.
- 라우터는 3계층에서 동작하며, 3계층에서 정의한 IP 주소를 이해할 수 있다.
    - IP 주소를 사용해 최적의 경로를 찾아주고 해당 경로로 패킷을 전송하는 역할을 한다.
        - 패킷이란?
            - **인터넷이나 네트워크를 통해 전송되는 데이터를 작게 나눈 기본 단위**를 뜻한다.

> 4계층 (트랜스포트 계층)


```yml
포트를 이용해 시스템 내의 응용 프로그램 간 통신을 가능하게 함
```

- 실제로 해당 데이터들이 정상적으로 잘 보내지도록 확인하는 역할을 한다
- 3계층에서 **패킷**을 전송하다보니(데이터를 분할해 패킷에 실어보내다 보니) **중간에 패킷이 유실되거나 순서가 바뀌는 경우**가 생길 수 있는데, **이 문제를 해결**하는 역할이 4계층이다.
    - 문제가 생기는 경우
        - 패킷 헤더에 보내는 순서와 받는 순서를 적어 통신하는데, 패킷에 **보내는 순서**를 명시한 것이 **시퀀스 번호, 받는 순서**를 명시한 것이 **ACK 번호**
        - 장치 내의 애플리케이션을 구분할 수 있도록 하는 것이 **포트 번호**
- 로드 밸런서와 방화벽이 있다.
    - _포트 번호, ACK, 시퀀스 번호 정보를 이용해 부하를 분산_하거나 보안 정책을 수립해 캐릿을 통과, 차단하는 기능을 수행한다.
        - _해당 번호들로 어떻게 부하를 분산하는가?_

> 5계층 (세션 계층)
> 

```yml
양 끝단의 응용 프로세스가 통신을 관리하는 방법을 제공하는 계층.
```

- 작업 완료 후에는 연결을 끊는 역할을 한다
- TCP/IP 세션을 만들고 없애는 책임을 진다.
- 에러로 중단된 통신에 대한 에러 복구와 재전송도 수행한다

> 6계층 (프레젠테이션 계층)
> 


```yml
코드 간 번역을 담당하는 계층
```

- 각기 다른 애플리케이션이나 시스템 간의 통신을 돕기 위해 하나의 통일된 구문 형식으로 변환시키는 기능을 수행한다.

> 7계층 (애플리케이션 계층)


```yml
응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행하는 계층
```

- 애플리케이션 프로세스를 정의하고 애플리케이션 서비스를 수행한다.
- 네트워크 SW의 UI 부분이나 사용자 입/출력 부분을 정의한다
- 애플리케이션 계층의 대표적 프로토콜 종류
    - FTP, SMTP, HTTP, TELNET

### 인캡슐레이션과 디캡슐레이션

```yml
- 인캡슐레이션: 애플리케이션에서 하위 계층(1~4)에 데이터를 보내면서 _패킷에 데이터를 넣을 수 있도록_ 분할하는 과정
- 디캡슐레이션: 하위 계층(1~4)에서 보낸 데이터를 받는 과정
```

1. 패킷은 무엇인가? 패킷은 pocket처럼 실체 없는 껍데기(포장지)이고 그 안에 데이터를 넣어야 하는건가?
2. 패킷 네트워크가 왜 회선 전체를 점유하지 않고 동시에 여러 단말이 통신하도록 해준다고 하는가?
3. 2계층에서 랜카드가, ‘2계층에 적힌 정보 중 목적지가 자신에게 온 것이 아니라면 해당 패킷을 버린다’라고 할 때 목적지 확인하는 역할을 수행하는 것인가? **(O)**
    1. _‘나에게 온 패킷이 맞다면 2계층의 헤더 정보는 제외하고 3계층으로 올려보낸다’고 했는데, 어떻게 2계층의 헤더 제외는 제외하는가?_
    2. _**2계층의 헤더 정보를 제외하고 3계층으로 올려보낸다고 했는데, 왜 라우터에 관한 설명을 할 때 ‘2계층의 헤더를 제외했다가, 목적지의 헤더를 다시 붙인다’라고 하는가?**_

- 인캡슐레이션 하는 과정에서 패킷에 데이터를 넣을 수 있도록 데이터를 분할하는데, 4계층부터는 네트워크 전송을 위한 정보를 헤더에 붙여 넣는다.
    - 헤더 정보는 4, 3, 2계층에서 각각 자신이 필요한 정보를 추가한다.
        - 이때 필요한 정보로 헤더에 들어가는 내용이 4계층: 포트, ack, 시퀀스 / 3계층: IP 주소 / 2계층: MAC 주소
    - 각 계층의 헤더 정보는 받는 상대방이 확인하는 정보이다. 즉 각 계층끼리 전달된다.
        - 4계층의 헤더는 그 정보를 받는 쪽의 4계층에서 확인한다.
    - 헤더에 넣는 정보는 너무나도 많지만, 다음 두 가지는 꼭 포함되어있어야한다.
        - **현재 계층에서 정의하는 정보**
            - TCP의 4계층 프로토콜인 TCP에선: 시퀀스, ack 번호
            - 3계층: 논리적인 주소인 출발지, 도착지 IP 주소
            - 2계층: 출발지, 도착지의 MAC 주소
        - **상위 프로토콜 지시자**
            - 디캡슐레이션 할 때(즉 하위 → 상위 계층) 상위 프로토콜이 너무 많은데 헤더에 아무런 정보가 없으면 어떤 상위 프로토콜로 올려보내줘야할지 모른다.
            - 따라서 애초에 인캡슐레이션 할 때 (상위 → 하위 계층) 인캡슐레이션 하는 쪽에서 헤더에 상위 프로토콜 지시자 정보를 포함해야 한다.
- _각 계층마다의 상위 프로토콜 지시자_
    - 4계층: 포트 번호
    - 3계층: 프로토콜 번호
    - 2계층: 이더 타입
        - _4계층의 상위 프로토콜 지시자를 나타내는 포트 번호는 그러면 5, 6, 7계층을 나타내는 말인가?_
        - _3계층은 4~7계층이고 2계층은 3~7계층인건가?_
