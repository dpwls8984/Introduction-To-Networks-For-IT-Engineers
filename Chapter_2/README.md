### 용어 정리

1. 이더넷
    1. 컴퓨터와 네트워크 장비를 **유선 케이블로 연결하여 데이터를 전송하는 LAN 기술**
    2. 현재 가장 널리 사용되는 유선 네트워크 방식 (네트워크 기술 전체 개념)
    3. 특정 물리적 네트워크 규격(케이블, 신호 방식 등)과
        
        그 위에서 데이터를 전달하는 규칙(MAC 주소, 프레임 구조 등)을
        
        함께 정의한 LAN 네트워크 표준이다
        
2. 이더넷 기반 네트워크
    1. 이더넷을 사용해 구축된 실제 네트워크
    2. 이더넷 기술을 사용해서 만든 LAN

### 네트워크 연결 구분

- LAN (Local Area Network)
    - 소규모 네트워크, 스위치 같이 간단한 장비로 연결된 네트워크
- WAN (Wide Area Network)
    - 먼 거리에 있는 네트워크 연결하거나 인터넷에 접속하기 위한 네트워크

### 네트워크 회선

1. 이더넷과 메트로 이더넷의 차이점은 무엇인가?
    1. Ethernet  →  LAN 기술
    Metro Ethernet → Ethernet 기술을 도시 규모로 확장한 네트워크
    2. 메트로 이더넷은 통신사가 제공하는 도시 단위 네트워크 서비스
    3. 즉 LAN 기술(Ethernet)을 MAN(Metropolitan Area Network)에 적용한 것
2. 이더넷 방식 전용선에서 왜 _LLCP_ 설정이 필수인가?
    1. _LLCP (Link loss carry forward) 란?_
        1. 링크 장애(Link Loss)가 발생하면 그 장애를 반대 방향으로 전달하여 양쪽 장비가 링크 다운을 인식하게 하는 기능
    2. _LLCP 설정을 해두지 않으면 전용 회선이 한 사이트에서 다운되어도 반대쪽 사이트에서는 정상이라고 인식하기 때문_
    
    ```plain text
    회사A Switch ── 통신사 장비1   X   통신사 장비2 ── 회사B Switch
    ```
    
    - 회사A 스위치는 **링크가 정상이라고 인식**
    - 회사B 스위치도 **링크가 정상이라고 인식**
        - 이런 상황을 Blackhole 상황이라고 함
        - 그래서 한쪽 링크가 끊어지면반대쪽 링크도 강제로 down 처리한다
3. MPLS VPN (Multiprotocol Label Switching Virtual Private Network) 이란 무엇인가? 
    - 통신사의 _**MPLS 네트워크를 이용해 기업의 여러 지점을 연결_하는 VPN 기술**
    - 서로 다른 고객의 네트워크를 **논리적으로 분리된 사설망**처럼 제공
        - 지점 Router ── 통신사 Router ── MPLS Core ── 통신사 Router ── 본사 Router

### 네트워크 구성 요소

- NIC는 컴퓨터를 네트워크에 연결하기 위한 하드웨어 장치이다.
    - NIC의 역할
        - 직렬화 (Serialization)
            - 전기적 신호 ↔ 데이터 신호 변환
            - 이런 상호 변환작업을 직렬화라고 한다
        - MAC 주소
        - 흐름 제어 (flow control)
- 허브
    - 케이블과 동일하게 1계층에서 동작하는 장비
    - 거리가 멀어질수록 줄어드는 전기 신호를 재생성해주고 HUB라는 용어 그대로 여러 대의 장비를 연결할 목적으로 사용된다
    - 현재는 거의 사용되고 있지 않다 (들어온 신호를 단순히 모든 포트로 내보내서 네트워크에 접속된 모든 단말이 경쟁하게 되어 성능이 줄어든다는 단점이 있음)
- 스위치
    - 여러 장비를 연결하고 통신을 중재하는 **2계층 장비**
    - 허브의 역할과 통신을 중재하는 2가지 역할을 모두 포함하므로 스위칭 허브라고도 불림
    - **MAC 주소를 이해**할 수 있어 목적지 MAC 주소의 위치를 파악하고 **목적지가 연결된 포트로만 전기 신호를 보낸다** (허브와의 차이점)
- 라우터
    - 3계층에서 동작하면서 먼 거리로 통신할 수 있는 _프로토콜로 변환_한다
        - _무엇을 프로토콜로 변환하는가? 프로토콜로 변환한다는 게 무엇인가?_
            - 데이터가 라우터에 도착하면, 라우터는 목적지 MAC 주소가 적힌 **L2(이더넷) 헤더**를 버린다. (**De-encapsulation**).
            - 그다음 L3(IP) 주소를 보고 다음 목적지를 알아낸다.
            - 다음 목적지로 가기 위해 필요한 새로운 **L2 헤더**를 다시 붙여서 보낸다.(**Re-encapsulation**)
    - 원격지로 쓸데없는 패킷이 전송되지 않도록 브로드캐스트와 멀티캐스트를 컨트롤하고 불문명한 주소로 통신을 시도할 경우 이를 버린다.
    - 정확한 방향으로 패킷이 전송되도록 경로를 지정하고 최적의 경로로 패킷을 포워딩한다
    - L3 스위치와 공유기 ⇒ 라우터와 비슷한 역할을 수행함
- 로드 밸런서
    - 4계층에서 동작
        - 애플리케이션 계층에서 애플리케이션 프로토콜의 특징을 이해하고 동작하는 7계층 로드 밸런서를 별도로 ADC (Application Delivery Controller)라고 부름.
        - L4 스위치라고 부르는 네트워크 장비로 _로드 밸런서의 한 종류_로, 스위치처럼 여러 포트를 가지고 있으면서 로드 밸런서 역할을 하는 장비를 지칭한다.
            - _L4 스위치는 4계층에서 동작하는 로드밸런서 맞는지? ADC가 아니라?_
                - L4 스위치는 4계층(TCP/UDP) 정보를 바탕으로 부하를 분산하는 하드웨어형 로드밸런서가 맞다.
                - **ADC**(Application Delivery Controller)는 이 L4/L7 로드밸런싱 기능을 포함하여 보안, 가속 등 더 많은 기능을 수행하는 **'로드밸런서의 진화된 상위 버전’**이다.
    - 4계층 포트 주소를 확인하는 동시에 IP 주소를 변경할 수 있다.
    - 웹 서버를 증설하고 싶을 때 _로드 밸런서를 웹 서버 앞에 두고_ 웹 서버를 여러 대로 늘려준다.
        - _웹 서버 앞에 둔다는 게 무슨 뜻인지?_
            - **"앞에 둔다"는 것은 물리적(하드웨어적) 배치와 논리적(개념적) 흐름을 모두 포함하는 말**로, 사용자의 요청이 웹 서버에 닿기 전에 **반드시 로드 밸런서를 먼저 거치게 만든다**는 뜻이다.
            - 개념적으로 앞에 둔다는 뜻
                - 서버들의 대표 IP를 로드 밸런서에게 주는 것이다. 사용자는 오직 로드 밸런서의 주소로만 접속하게 하고, 로드 밸런서가 요청을 받아서 뒤에 있는 서버들에게 전달(Proxy)해주는 구조
    - 대표 IP는 로드 밸런서가 갖고 로드 밸런서가 각 웹 서버로 패킷의 목적지 IP 주소를 변경해 보내준다.
- 보안 장비 (방화벽/IPS)
    - 방화벽은 4계층에서 동작해 방화벽을 통과하는 _패킷의 3, 4계층 정보를 확인_하고 패킷의 정책과 비교해 버리거나 포워딩한다.
        - _방화벽은 4계층에서 동작하는데 왜 패킷의 3, 5계층 정보가 아니라 3, 4계층 정보를 확인하는가?_
            - **"누가(IP) 어떤 서비스(Port)를 요청하는가"**가 보안의 가장 기본이 되기 때문
            - 5계층(세션) 이상은 '데이터의 내용'에 해당하는데, 이를 확인하려면 훨씬 많은 비용과 시간이 든다. 그 안에 내용을 확인하는 건 다른 전문 보안 장비의 역할이다.
            - 방화벽 정책(Rule)의 핵심은 **5-Tuple**이라 불리는 정보입니다.
                - **3계층 정보:** 출발지 IP, 목적지 IP
                - **4계층 정보:** 출발지 Port, 목적지 Port, 프로토콜(TCP/UDP)
            - 서버를 띄울 때 패킷의 헤더에 들어있는 목적지 포트가 열려있기만 하면(’정상적인 3, 4계층 절차(TCP Handshake 등)를 거쳤는가’만 확인하고) 통과시키는 것이다.
